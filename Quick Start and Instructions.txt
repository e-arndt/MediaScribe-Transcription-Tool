MediaScribe - Quick Start / How to Use

MediaScribe creates transcripts and optional caption files from audio and video files.

It includes:

1. The MediaScribe GUI - recommended for most users
2. The MediaScribe Terminal version - useful for advanced users, fallback, development, and troubleshooting

MediaScribe is especially useful after old family recordings have been digitized, such as VHS, VHS-C, 8mm film, Super 8 film, Video8, Hi8, audio cassette, reel-to-reel recordings, talks, interviews, family stories, lessons, meetings, and other spoken audio/video files.

============================================================
1. INSTALL MEDIASCRIBE
============================================================

Open the folder where you downloaded, extracted, or copied MediaScribe.

This might be a USB drive, Downloads folder, Desktop folder, project folder, or extracted ZIP folder.

Double-click:

Install.bat

The installer starts PowerShell and runs setup.ps1.

During setup, choose where to install MediaScribe.

The default install folder is usually:

Documents\MediaScribe

Setup creates the installed MediaScribe folder and these subfolders:

Input
Output
Logs
Models
Tools

Setup also creates launchers for the GUI and Terminal versions.

The main GUI launcher is:

Start-MediaScribe-GUI.bat

The Terminal launcher may be:

MediaScribe.bat

or:

Start-MediaScribe.bat

The exact launcher names may vary slightly by package version.

If setup completes successfully, it may offer choices such as:

[1] Start MediaScribe GUI
[2] Start MediaScribe Terminal
[3] Open Instructions
[Q] Quit setup

Choose the GUI for normal use.

============================================================
2. REINSTALL OR REPAIR MEDIASCRIBE
============================================================

Run Install.bat again to reinstall or repair MediaScribe.

This is useful when core app files were damaged, deleted, or updated.

Setup may replace core files such as:

transcribe.ps1
MediaScribe-GUI.ps1
README.md
Quick Start and Instructions.txt
MediaScribe.bat
Start-MediaScribe-GUI.bat
config.json

Setup recreates required folders if they are missing.

Setup does not clear or delete files inside:

Input
Output
Logs
Models
Tools

Existing media files, transcripts, WAV files, captions, logs, and models should remain in place.

To uninstall MediaScribe, delete the installed MediaScribe folder.

============================================================
3. DEPENDENCIES
============================================================

MediaScribe requires:

Python
pip
OpenAI Whisper
FFmpeg

The setup script checks these dependencies.

FFmpeg may be bundled in:

Dependencies\FFmpeg

During setup, bundled FFmpeg is copied to:

Tools\ffmpeg

MediaScribe looks for FFmpeg in this order:

1. Local Tools\ffmpeg\ffmpeg.exe
2. System FFmpeg available through PATH

Python may be bundled in:

Dependencies\Python

If Python is missing, setup can offer to run the bundled Python installer.

If pip is missing, setup can try to repair it using Python's ensurepip feature.

OpenAI Whisper is installed with pip when it is missing.

Installing Whisper requires internet access.

MediaScribe looks for Whisper in this order:

1. python -m whisper
2. Global whisper.exe or whisper command

If setup reports a missing dependency, MediaScribe files may still be installed, but transcription will not work until the dependency is installed or repaired.

After fixing the dependency or internet connection, run Install.bat again.

4. START THE MEDIASCRIBE GUI
============================================================

Open the installed MediaScribe folder.

Example:

Documents\MediaScribe

Double-click:

Start-MediaScribe-GUI.bat

The GUI is recommended for most users.

The GUI includes:

Files
Files to transcribe
Transcription Settings
Status
Start Transcription / Start Batch Transcription
Stop Transcription
Open Output Folder
Clear Status Window
Close MediaScribe

The Status section expands when the window is enlarged. The upper controls remain in their normal positions, and the Close button stays centered near the bottom.

============================================================
5. SUPPORT WINDOW AND SINGLE-INSTANCE PROTECTION
============================================================

A minimized PowerShell support window may appear on the taskbar.

Its title may say:

DO NOT CLOSE - MediaScribe Support Window

Do not close this support window while MediaScribe is running.

Close MediaScribe from the GUI when finished.

Only one MediaScribe GUI instance can run at a time in the current Windows logon session.

If you try to start another GUI instance, MediaScribe displays:

MediaScribe is already running.

Check the taskbar for the open MediaScribe window.

The duplicate GUI instance then exits.

The Terminal version remains separately available for advanced use and troubleshooting.

============================================================
6. CHOOSE A SOURCE FOLDER
============================================================

The GUI uses a Source folder.

By default, the Source folder is the MediaScribe Input folder.

Place audio/video files in Input before starting MediaScribe, or use:

Browse Folder

to choose another folder, such as a USB drive, Downloads folder, Desktop folder, or external drive.

Use:

Refresh Files

to rescan the current Source folder.

Use:

Open Source Folder

to open the Source folder in File Explorer.

Recognized video files:

MP4, MKV, MOV, M4V, AVI, WEBM

Recognized audio files:

MP3, M4A, WAV, AAC, FLAC, OGG, OPUS, WMA

============================================================
7. CHOOSE ONE FILE OR ALL FILES
============================================================

Under Files to transcribe, choose:

Selected file

or:

All files in source folder

------------------------------------------------------------
Selected file
------------------------------------------------------------

Use Selected file to process one recording.

Choose the recording from the Selected file dropdown.

This is the default mode.

------------------------------------------------------------
All files in source folder
------------------------------------------------------------

Use All files in source folder to process every recognized media file in the current Source folder.

MediaScribe captures the recognized file list and processes files one at a time.

It does not run several Whisper jobs at once.

The same settings are used for every file in the batch:

Output mode
Transcription mode
Input Audio
Output Text

Each file receives its own output folder.

If one file fails, MediaScribe continues with the remaining files and reports the failure in the final batch summary.

After the batch finishes or stops, the Source list refreshes.

============================================================
8. GUI TRANSCRIPTION SETTINGS
============================================================

The GUI has four main settings.

------------------------------------------------------------
Output mode
------------------------------------------------------------

Default - Original file, WAV audio, TXT transcript

Default mode is recommended for most users.

Full - Original file, WAV audio, TXT, SRT, VTT, TSV, JSON

Use Full mode when you need caption/subtitle files or detailed Whisper output.

SRT and VTT files are useful for video captions and subtitles.

------------------------------------------------------------
Transcription mode
------------------------------------------------------------

Fast - recommended

Fast uses the configured/default Whisper model, normally medium.

Accurate - slower, for difficult audio

Accurate uses the large Whisper model. It may help with harder audio but requires more time and computer resources.

------------------------------------------------------------
Input Audio
------------------------------------------------------------

Input Audio means the language spoken in the recording.

Options include:

English
Auto-detect
Chinese
French
German
Italian
Japanese
Korean
Portuguese
Spanish

English is the default.

Use Auto-detect when you do not know the spoken language or when a batch contains different languages.

Auto-detect evaluates every file independently.

Example mixed-language batch:

English recording -> detected as English
Korean recording -> detected as Korean
Spanish recording -> detected as Spanish

Choosing English tells Whisper to treat every file as English. Non-English recordings may then produce inaccurate or unusable text.

------------------------------------------------------------
Output Text
------------------------------------------------------------

Same as input / detected

Creates a transcript in the same language as the audio.

Examples:

English audio -> English transcript
Korean audio -> Korean transcript
Auto-detect Korean audio -> Korean transcript

English translation

Asks Whisper to translate the spoken audio into English.

Examples:

Korean audio -> English text
Spanish audio -> English text
Auto-detect Korean audio -> English text

Whisper's built-in translation mode translates to English. MediaScribe does not currently translate into every possible output language.

Recommended mixed-language settings:

To keep each recording in its detected language:

Input Audio: Auto-detect
Output Text: Same as input / detected

To create English text from each recording:

Input Audio: Auto-detect
Output Text: English translation

============================================================
9. LIVE TRANSCRIPT PREVIEW
============================================================

For English transcription, MediaScribe shows live transcript lines while Whisper is working when:

Input Audio: English
Output Text: Same as input / detected

Whisper normally releases the preview in approximately 30-second audio chunks.

This means several timestamped lines may appear together after each chunk is processed.

This is normal and confirms that progress is being released while the transcription is still running.

Live preview is hidden for:

Auto-detect
Non-English input languages
English translation

This is intentional.

Some languages use characters that may not display correctly in the Windows status window.

When preview is hidden, MediaScribe still creates the final transcript.

The GUI or Terminal may show messages such as:

Live transcript preview is not shown for this language/output setting.
Some characters may not display correctly in the Windows status window.
English translation and non-English transcription still create the final transcript file.

Hidden-preview jobs may also create:

whisper_stdout.log
whisper_stderr.log

These logs are normal.

============================================================
10. START TRANSCRIPTION
============================================================

After choosing the file mode and settings, click:

Start Transcription

When All files in source folder is selected, the button changes to:

Start Batch Transcription

For every file, MediaScribe:

1. Creates a separate output/job folder
2. Extracts mono 16 kHz WAV audio with FFmpeg
3. Runs OpenAI Whisper
4. Creates the selected transcript/output files
5. Verifies expected output
6. Moves or copies the original according to the file-handling rules
7. Continues to the next file in batch mode

Long recordings may take several minutes or more.

Accurate mode is slower than Fast mode.

Do not close the GUI or its PowerShell support window during transcription.

Use Stop Transcription when you need to cancel safely.

============================================================
11. STOP TRANSCRIPTION IN THE GUI
============================================================

The Stop button is disabled while MediaScribe is idle.

During transcription, Stop becomes active and uses the same red attention color as the MediaScribe section titles.

To stop:

1. Click Stop Transcription
2. Read the confirmation
3. Choose Yes only when you want to stop

The safe/default choice is:

No, Keep Running

After Stop is confirmed, the button changes to:

Stopping...

MediaScribe stops the active FFmpeg, Python, and Whisper process tree and returns the GUI to an idle state.

------------------------------------------------------------
Stopping one selected file
------------------------------------------------------------

The active transcription is stopped.

The original source media remains unchanged.

The incomplete output/job folder is renamed with an ABORTED_ prefix and timestamp.

Example:

Output\ABORTED_Family Interview_20260723_205808

------------------------------------------------------------
Stopping a batch
------------------------------------------------------------

Stop cancels the entire active batch.

Completed files remain completed.

The active file is stopped and its incomplete job folder becomes ABORTED_.

Files that have not started remain untouched in the Source folder.

The summary reports:

Files found
Completed
Failed
Stopped
Not started

A stopped job is a warning/attention condition, not a processing failure.

Closing MediaScribe remains blocked while transcription is active.

Use Stop first, then close MediaScribe after it returns to idle.

============================================================
12. FIND RESULTS
============================================================

Results are saved in Output.

Each file receives its own output folder.

Example:

Output\My Recording

Default mode normally contains:

My Recording.mp4
My Recording.wav
My Recording.txt

Full mode may contain:

My Recording.mp4
My Recording.wav
My Recording.txt
My Recording.srt
My Recording.vtt
My Recording.tsv
My Recording.json

Hidden-preview jobs may also contain:

whisper_stdout.log
whisper_stderr.log

If a folder with the same name already exists, MediaScribe creates a timestamped sibling folder instead of overwriting the existing job.

If a transcription is stopped, the incomplete folder receives an ABORTED_ prefix.

============================================================
13. FILE HANDLING AND SAFETY
============================================================

Files inside the MediaScribe Input folder are moved into their output folder only after successful processing and expected output verification.

External files selected from another folder are left in place by default.

The original is never moved before the expected transcript output is created.

For every file:

Successful Input-folder file -> moved into completed job folder
Failed Input-folder file -> remains in Input
Stopped active file -> remains in Source/Input
Unstarted batch file -> remains untouched
External source file -> remains in its external folder by default

These rules are applied separately to every file in a batch.

============================================================
14. FAILURES AND INCOMPLETE FOLDERS
============================================================

DRM-protected, unsupported, damaged, or incorrectly named files may fail.

A failed file is reported as a failure, not as stopped.

In a batch, MediaScribe continues to later files after an individual failure.

The original failed source remains in place.

When FFmpeg fails before creating anything useful, MediaScribe removes the empty failed job folder automatically.

MediaScribe does not delete a failed folder when it contains useful or diagnostic content, such as:

Partial WAV audio
Whisper logs
Transcript fragments
Other output files
Hidden files

ABORTED_ folders are used only for user-requested Stop operations.

An ABORTED_ folder may contain useful incomplete work and can be reviewed, retained, or deleted.

============================================================
15. USE THE TERMINAL VERSION
============================================================

The GUI is recommended for most users.

The Terminal version is available for advanced users, development, fallback, and troubleshooting.

Open the installed MediaScribe folder and double-click:

MediaScribe.bat

or:

Start-MediaScribe.bat

The Terminal version displays a welcome screen and the Input/Output locations.

Press Enter to continue.

When recognized files are found, it displays a menu such as:

[1] First file.mp4
[2] Second file.mp3
[B] Process all files
[Q] Quit

Choose a file number to process one file.

Choose B to process all recognized files.

Choose Q to quit.

If no files are found:

[R] Refresh / check again
[O] Open Input folder
[Q] Quit

Pressing Enter refreshes.

============================================================
17. STOP TRANSCRIPTION IN THE TERMINAL
============================================================

While the Terminal version is processing a file, it displays a yellow attention option:

[S] Stop transcription safely

For a batch, it displays:

[S] Stop transcription safely and cancel the remaining batch

Press S to request a safe Stop.

MediaScribe displays a confirmation:

[Y] Yes, stop
[N] No, keep running
Default: No, keep running

Press Y to stop.

Press N or Enter to continue processing.

A confirmed Stop remains a stopped operation even when FFmpeg or Whisper happens to finish while the confirmation is being answered.

In single-file mode:

The active original remains unchanged.
The incomplete folder becomes ABORTED_.
The Terminal returns to the Next Step menu.

In batch mode:

Completed files remain completed.
The active file becomes stopped.
The active original remains unchanged.
The active incomplete folder becomes ABORTED_.
Remaining files do not start.
The final summary reports Completed, Failed, Stopped, and Not started.

Use the controlled S option instead of Ctrl+C whenever possible.

============================================================
18. TERMINAL COLORS AND EXIT CODES
============================================================

The Terminal interface uses colors to communicate status:

Green:
[OK]
Successful file or completed count

Yellow:
[WARN]
[STOPPED]
[NOT STARTED]
Attention, warning, stopped, or unstarted work

Red:
[ERROR]
[FAILED]
Actual processing failure

General information uses the normal console color.

Exit codes:

0 = completed successfully
1 = completed with one or more errors
2 = stopped by the user

In a stopped batch, exit code 2 takes priority even when an earlier file failed.

============================================================
19. AFTER PROCESSING
============================================================

After processing, the Terminal version asks:

[R] Run again / return to file list
[O] Open output folder
[Q] Quit

Pressing Enter quits.

The GUI refreshes the current Source list after a single job or batch.

Files successfully moved from Input disappear from the list.

Failed, stopped, and unstarted Input-folder files remain listed because they remain in Input.

External source files remain listed because their originals stay in the external Source folder.

============================================================
21. TROUBLESHOOTING
============================================================

A second GUI will not open:

This is intentional. Check the taskbar for the already-open MediaScribe window.

No live preview appears:

Live preview is intentionally hidden for Auto-detect, non-English input, and English translation. The final transcript is still created.

Preview appears in groups:

Whisper normally releases progress in approximately 30-second audio chunks.

FFmpeg shows decoder warnings but processing completes:

The source may contain unusual or damaged frames that FFmpeg recovered from. Review the resulting WAV and transcript.

A file fails immediately:

The file may be damaged, unsupported, DRM-protected, or may have the wrong extension. The original remains in place.

An empty output folder disappears after failure:

This is intentional. MediaScribe removes only a truly empty failed folder.

An ABORTED_ folder remains:

This records incomplete work from a confirmed Stop. Review or delete it when appropriate.

MediaScribe says a dependency is missing:

Repair Python, pip, Whisper, or FFmpeg, then run Install.bat again.

Default mode is recommended for most users.

Full mode is useful for caption and subtitle workflows.

Input Audio means the language spoken in the recording.

Output Text controls whether final text stays in the original/detected language or is translated to English.

Multiple-file processing is sequential, not parallel.

Use Auto-detect for mixed-language batches.

English live preview appears in approximately 30-second chunks.

Stop safely cancels the active process tree.

Completed batch files remain completed.

Failed, stopped, and unstarted originals remain in place.

ABORTED_ folders identify incomplete user-stopped jobs.

Install.bat is used to install, reinstall, or repair MediaScribe.

After installation, use the GUI launcher or Terminal launcher from the installed MediaScribe folder.
