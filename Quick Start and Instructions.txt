MediaScribe - Quick Start / How to Use

MediaScribe creates transcripts and optional caption files from audio and video files.

It can be used with:

1. The MediaScribe GUI - recommended for most users
2. The MediaScribe Terminal version - useful as a fallback or for troubleshooting

MediaScribe is especially useful after old family recordings have been digitized, such as VHS, VHS-C, 8mm film, Super 8 film, Video8, Hi8, audio cassette, reel-to-reel recordings, talks, interviews, family stories, lessons, meetings, and other spoken audio/video files.

============================================================
1. INSTALL MEDIASCRIBE
============================================================

Open the folder where you downloaded, extracted, or copied MediaScribe.

This might be a USB drive, Downloads folder, Desktop folder, project folder, or extracted ZIP folder.

In that folder, double-click:

Install.bat

The installer will start PowerShell and run setup.ps1 for you.

During setup, choose where to install MediaScribe.

The default install folder is usually:

Documents\MediaScribe

Setup will create the installed MediaScribe folder and these subfolders:

Input
Output
Logs
Models
Tools

Setup will also create launchers for using MediaScribe.

The main GUI launcher is:

Start-MediaScribe-GUI.bat

The terminal launcher is:

MediaScribe.bat

or:

Start-MediaScribe.bat

The exact launcher names may vary slightly depending on the package version, but the GUI launcher is the recommended choice for most users.

If setup completes successfully and all required dependencies are available, the installer may offer launch options such as:

[1] Start MediaScribe GUI
[2] Start MediaScribe Terminal
[3] Open Instructions
[Q] Quit setup

Choose the GUI for normal use.

============================================================
2. REINSTALLING OR REPAIRING MEDIASCRIBE
============================================================

You can run Install.bat again to reinstall or repair MediaScribe.

This is useful if core app files were damaged, deleted, or need to be updated.

During reinstall or repair, setup may replace the core app files, including:

transcribe.ps1
MediaScribe-GUI.ps1
README.md
Quick Start and Instructions.txt
MediaScribe.bat
Start-MediaScribe-GUI.bat
config.json

Setup recreates required folders if they are missing.

Setup does not clear or delete files inside these folders:

Input
Output
Logs
Models
Tools

This means existing media files, transcripts, WAV files, caption files, and logs should remain in place.

To remove MediaScribe, delete the installed MediaScribe folder.

============================================================
3. DEPENDENCIES
============================================================

MediaScribe requires:

Python
pip
OpenAI Whisper
FFmpeg

The setup script checks for these dependencies.

FFmpeg may be bundled with MediaScribe in:

Dependencies\FFmpeg

During setup, bundled FFmpeg is copied into the installed app folder here:

Tools\ffmpeg

MediaScribe looks for FFmpeg in this order:

1. Local Tools\ffmpeg\ffmpeg.exe
2. System FFmpeg from PATH

Python may be bundled with MediaScribe in:

Dependencies\Python

If Python is missing, setup can offer to run the bundled Python installer.

pip is handled through Python. If pip is missing, setup can try to repair it using Python's ensurepip feature.

OpenAI Whisper is installed with pip if it is missing.

Installing Whisper requires internet access.

MediaScribe looks for Whisper in this order:

1. python -m whisper
2. Global whisper.exe / whisper command

If setup says a dependency is missing, MediaScribe files may still be installed, but transcription will not work until the missing dependency is installed or fixed.

After fixing missing dependencies or internet access, run Install.bat again to re-check setup.

============================================================
4. USE MEDIASCRIBE GUI
============================================================

After setup finishes, choose the GUI launch option if offered.

To use MediaScribe later, open the installed MediaScribe folder.

Example:

Documents\MediaScribe

Then double-click:

Start-MediaScribe-GUI.bat

The GUI is recommended for most users.

When the GUI opens, it shows:

Files
Files to transcribe
Transcription Settings
Status
Start Transcription / Start Batch Transcription
Stop Transcription
Open Output Folder
Clear Status Window
Close MediaScribe

A minimized PowerShell support window may also appear on the taskbar.

Its title may say:

DO NOT CLOSE - MediaScribe Support Window

Do not close the support window while MediaScribe is running.

Close MediaScribe from the GUI when you are finished.

============================================================
5. CHOOSE A SOURCE FOLDER
============================================================

The GUI uses a Source folder.

By default, the Source folder is the MediaScribe Input folder.

You can put audio/video files into the Input folder before starting MediaScribe.

You can also use:

Browse Folder

to choose another folder, such as a USB drive, Downloads folder, Desktop folder, or external drive.

Use:

Refresh Files

to rescan the current Source folder.

Use:

Open Source Folder

to open the current Source folder in File Explorer.

Recognized video files:

MP4, MKV, MOV, M4V, AVI, WEBM

Recognized audio files:

MP3, M4A, WAV, AAC, FLAC, OGG, OPUS, WMA

============================================================
6. CHOOSE ONE FILE OR ALL FILES
============================================================

The GUI can process either one selected file or all recognized files in the current Source folder.

Under Files to transcribe, choose:

Selected file

or:

All files in source folder

------------------------------------------------------------
Selected file
------------------------------------------------------------

Use Selected file when you want to process only one recording.

Choose the recording from the Selected file dropdown.

This remains the default behavior.

------------------------------------------------------------
All files in source folder
------------------------------------------------------------

Use All files in source folder when you want MediaScribe to process every recognized audio/video file in the current Source folder.

MediaScribe captures the file list and processes the files one at a time.

It does not run several Whisper jobs at the same time.

The same Output mode, Transcription mode, Input Audio, and Output Text settings are used for every file in the batch.

Each file receives its own output folder.

If one file fails, MediaScribe should continue with the remaining files and report the results in the final batch summary.

After the batch finishes, the file list refreshes to match the current Source folder.

============================================================
7. TRANSCRIPTION SETTINGS IN THE GUI
============================================================

The GUI has four main settings.

------------------------------------------------------------
Output mode
------------------------------------------------------------

Default - Original file, WAV audio, TXT transcript

This is recommended for most users.

Default mode creates:

Original file
WAV audio
TXT transcript

Full - Original file, WAV audio, TXT, SRT, VTT, TSV, JSON

Use Full mode when you need caption/subtitle files or detailed Whisper output.

Full mode creates:

Original file
WAV audio
TXT transcript
SRT subtitle file
VTT subtitle file
TSV timing file
JSON Whisper output

SRT and VTT files are useful for video captions/subtitles.

------------------------------------------------------------
Transcription mode
------------------------------------------------------------

Fast - recommended

This is recommended for most users.

Fast uses the configured/default Whisper model.

Accurate - slower, for difficult audio

Accurate uses the large Whisper model.

It is slower, but may help when audio is harder to understand or Fast mode misses words.

------------------------------------------------------------
Input Audio
------------------------------------------------------------

Input Audio means the language being spoken in the recording.

Options include:

English
Auto-detect
Chinese
French
German
Italian
Japanese
Korean
Portuguese
Spanish

English is the default.

Use Auto-detect if you are not sure what language is being spoken.

If you know the language, choosing it directly may give more consistent results.

Important for multiple-file transcription:

The selected Input Audio setting is applied to every file in the batch.

Use Auto-detect when the folder contains recordings in different languages.

Auto-detect evaluates each file independently.

Example mixed-language batch:

English recording -> detected as English
Korean recording -> detected as Korean
Spanish recording -> detected as Spanish

If you select English instead of Auto-detect, Whisper is told to treat every file as English. A Korean or Spanish recording may then produce inaccurate or unusable text.

------------------------------------------------------------
Output Text
------------------------------------------------------------

Output Text controls what kind of text MediaScribe creates.

Same as input / detected

This creates a transcript in the same language as the audio.

Examples:

English audio -> English transcript
Korean audio -> Korean transcript
Auto-detect Korean audio -> Korean transcript

English translation

This asks Whisper to translate the spoken audio into English.

Examples:

Korean audio -> English text
Spanish audio -> English text
Auto-detect Korean audio -> English text

Important:

Whisper's built-in translation mode translates to English.

MediaScribe does not currently translate into every possible output language.

For a mixed-language batch:

Input Audio: Auto-detect
Output Text: Same as input / detected

creates each transcript in its detected language.

Input Audio: Auto-detect
Output Text: English translation

detects each source language and creates English output.

============================================================
8. LIVE PREVIEW BEHAVIOR
============================================================

For English transcription, MediaScribe can show live transcript preview lines while Whisper is working.

Live preview is shown when:

Input Audio: English
Output Text: Same as input / detected

Live preview is hidden for:

Auto-detect
Non-English input languages
English translation

This is intentional.

Some languages use characters that may not display correctly in the Windows status window.

When live preview is hidden, MediaScribe still creates the final transcript file.

The GUI may show messages such as:

Live transcript preview is not shown for this language/output setting.
Some characters may not display correctly in the Windows status window.
English translation and non-English transcription still create the final transcript file.

This is normal.

============================================================
9. START TRANSCRIPTION
============================================================

After choosing the file mode and settings, click:

Start Transcription

When All files in source folder is selected, the button changes to:

Start Batch Transcription

For Selected file mode, MediaScribe processes the file shown in the Selected file dropdown.

For All files in source folder mode, MediaScribe processes all recognized files in the current Source folder one at a time.

For each file, MediaScribe will:

1. Create a separate output/job folder
2. Extract WAV audio with FFmpeg
3. Run OpenAI Whisper
4. Create the selected transcript/output files
5. Move or copy the original file according to MediaScribe's file handling rules
6. Continue to the next file when using multiple-file mode

Long recordings may take several minutes or more.

Accurate mode is slower than Fast mode.

Do not close MediaScribe or its PowerShell support window while transcription is active.

Use Stop Transcription when you need to cancel the active operation safely.

============================================================
10. STOP TRANSCRIPTION
============================================================

Stop Transcription safely cancels the active GUI transcription.

The Stop button is disabled and uses its normal inactive appearance while MediaScribe is idle.

While transcription is running, the Stop button becomes active and uses the same red color as the MediaScribe section titles.

To stop:

1. Click Stop Transcription
2. Read the confirmation message
3. Choose Yes only when you want to cancel the active operation

The safe/default confirmation choice is No, Keep Running.

After Stop is confirmed, the button changes to:

Stopping...

MediaScribe then stops the active FFmpeg, Python, and Whisper process tree and returns the GUI to a usable idle state.

------------------------------------------------------------
Stopping one selected file
------------------------------------------------------------

In Selected file mode, Stop cancels only the active transcription.

The original source media remains unchanged.

The incomplete output/job folder is renamed with an ABORTED_ prefix.

Example:

Output\ABORTED_Family Interview_20260723_205808

------------------------------------------------------------
Stopping a multiple-file batch
------------------------------------------------------------

In All files in source folder mode, Stop cancels the entire active batch.

Completed files remain completed.

The active file is stopped and its incomplete job folder is renamed with ABORTED_.

Files that have not started remain untouched in the Source folder.

The stopped-batch summary reports:

Files found
Completed
Failed
Stopped
Not started

The user can remove unwanted source files, review or delete ABORTED_ folders, and restart the folder-wide batch.

------------------------------------------------------------
ABORTED_ folders
------------------------------------------------------------

An ABORTED_ folder identifies an incomplete job.

It may contain useful partial files, such as:

WAV audio
Whisper logs
Partial output files

The original source media is not renamed or moved when the active transcription is stopped.

ABORTED_ folders may be reviewed, retained, or deleted by the user, library staff, or a Family History Center technician.

Closing MediaScribe remains blocked while transcription is active.

Use Stop Transcription first, then close MediaScribe after it returns to an idle state.

============================================================
11. FIND RESULTS
============================================================

Results are saved in the Output folder.

Each file gets its own output folder.

Example:

Output\My Recording

In Default mode, the job folder contains:

My Recording.mp4
My Recording.wav
My Recording.txt

In Full mode, the job folder may contain:

My Recording.mp4
My Recording.wav
My Recording.txt
My Recording.srt
My Recording.vtt
My Recording.tsv
My Recording.json

For Auto-detect, non-English, or English translation jobs, the folder may also contain:

whisper_stdout.log
whisper_stderr.log

These log files are normal.

They keep Whisper's progress/debug output out of the GUI status window.

If an output folder with the same name already exists, MediaScribe creates a timestamped sibling folder instead of overwriting the existing job.

If a transcription is stopped, the incomplete job folder is renamed with an ABORTED_ prefix.

============================================================
12. MULTIPLE-FILE RESULTS
============================================================

In multiple-file mode, every recording receives its own output folder.

Example Source folder:

Family Interview.mp4
Korean Story.mp3
Grandpa Tape.wav

Example Output folder:

Output\Family Interview
Output\Korean Story
Output\Grandpa Tape

MediaScribe processes the files sequentially.

A completed file remains completed before the next file begins.

At the end, MediaScribe displays a batch summary showing which files completed and which files failed.

If the user stops a batch, the summary also reports the stopped file and the number of files that did not start.

After the batch finishes, the GUI refreshes the current Source folder list.

Files successfully moved from the MediaScribe Input folder disappear from the list.

External files normally remain listed because their originals remain in the external Source folder.

============================================================
13. FILE HANDLING
============================================================

Files placed inside the MediaScribe Input folder are moved into their output/job folder only after successful processing.

External files selected from another folder are left in place by default.

This behavior helps preserve original media while keeping completed jobs organized.

The original file is moved only after the expected transcript output is created.

If transcription fails, the original file remains in place.

During multiple-file transcription, these rules are applied separately to every file.

A successful Input-folder file is moved after that file succeeds.

A failed Input-folder file remains in the Input folder.

An external source file remains in its external folder by default.

When Stop Transcription is used, the active original media remains in place and the incomplete job folder is marked with ABORTED_.

14. USE MEDIASCRIBE TERMINAL VERSION
============================================================

The GUI is recommended for most users.

The terminal version is still available as a fallback.

To use the terminal version, open the installed MediaScribe folder and double-click:

MediaScribe.bat

or:

Start-MediaScribe.bat

The terminal version shows a welcome screen, then lets you choose files from the Input folder.

When MediaScribe starts, it shows:

What MediaScribe does
Default output information
Full output information
Recognized video formats
Recognized audio formats
Input folder location
Output folder location

Press Enter to continue.

MediaScribe will show a list of recognized files.

Example:

[1] First file.mp4
[2] Second file.mp3
[B] Process all files
[Q] Quit

Choose a file number to process one file.

Choose B to process all files.

Choose Q to quit.

If no files are found, you can add files to the Input folder and choose Refresh.

[R] Refresh / check again
[O] Open Input folder
[Q] Quit

Pressing Enter refreshes.

============================================================
15. TERMINAL OUTPUT MODES
============================================================

The terminal version asks what output files you want.

[D] Default - Original file, WAV audio, TXT transcript
[F] Full    - Original file, WAV audio, TXT, SRT, VTT, TSV, JSON

Default mode is recommended for most users.

Use Full mode when you need caption/subtitle files or detailed Whisper output.

============================================================
16. TERMINAL TRANSCRIPTION MODES
============================================================

The terminal version asks which transcription mode to use.

[F] Fast     - Uses the configured/default Whisper model
[A] Accurate - Uses the large Whisper model

Pressing Enter uses Fast mode.

Fast mode is recommended for most users.

Accurate mode is slower and may help when audio is harder to understand or Fast mode misses words.

============================================================
17. AFTER PROCESSING
============================================================

After processing finishes, MediaScribe may ask what to do next.

[R] Run again / return to file list
[O] Open output folder
[Q] Quit

Pressing Enter quits.

18. COMMON FAMILY HISTORY USES
============================================================

MediaScribe can help create searchable written transcripts from digitized family media, such as:

Family interviews
Home movie narration
Oral histories
Family stories
Funeral talks
Memorial recordings
Family gatherings
Personal memories
Church talks
Lessons
Discussions
Meetings
Dictation
Other spoken recordings

A common single-file workflow is:

1. Digitize the tape, film, or recording into a video/audio file
2. Open MediaScribe GUI
3. Choose the folder that contains the digitized file
4. Choose Selected file
5. Choose the file
6. Choose the language spoken in the recording
7. Choose whether to keep the original-language transcript or translate to English
8. Start transcription
9. Save or share the TXT transcript

A common multiple-file workflow is:

1. Place several digitized recordings into one folder
2. Open MediaScribe GUI
3. Select that Source folder
4. Choose All files in source folder
5. Choose Auto-detect when the files may contain different languages
6. Choose whether to keep each detected language or translate all output to English
7. Start transcription
8. Review the separate output folder created for every file
9. Review the final batch summary

============================================================
19. NOTES
============================================================

DRM/protected media may fail.

Damaged media files may fail.

Audio is extracted only and converted to mono 16 kHz WAV for consistency.

Default mode is recommended for most users.

Full mode is useful for caption/subtitle workflows.

Input Audio means the language spoken in the recording.

Output Text controls whether the final text is in the original/detected language or translated to English.

The same settings are applied to every file in a multiple-file batch.

Use Auto-detect for a mixed-language folder.

Multiple-file mode processes files one at a time and creates a separate output folder for each file.

Stop Transcription safely cancels the active GUI operation.

In batch mode, Stop cancels the active file and all files that have not yet started.

Completed files remain completed.

The active original media remains unchanged.

ABORTED_ folders contain incomplete work and may be reviewed or deleted.

Install.bat is only used to install, reinstall, or repair MediaScribe.

After installation, use the GUI launcher or MediaScribe.bat from the installed MediaScribe folder.
